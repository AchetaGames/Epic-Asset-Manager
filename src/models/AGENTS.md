# MODELS — Data Layer & GObject Wrappers

## OVERVIEW

GObject data models wrapping Epic/Fab API types + Diesel ORM for SQLite persistence. Central `Model` struct holds app-wide state.

## STRUCTURE

```
models/
├── mod.rs           # Model struct: EpicGames API client, settings, secrets, async_channel
├── database.rs      # r2d2 connection pool, Diesel migrations, db path resolution
├── asset_data.rs    # GObject wrapper for Epic Store AssetInfo (favorite, downloaded state)
├── engine_data.rs   # GObject wrapper for UE installations (version parsing, update checks)
├── project_data.rs  # GObject wrapper for .uproject files (async thumbnail loading)
├── plugin_data.rs   # GObject wrapper for .uplugin files (JSON metadata parsing)
├── category_data.rs # Lightweight GObject for sidebar category filters
└── log_data.rs      # GObject for log file entries (path, name, crash flag)
```

**Note**: No `fab_data.rs` yet — to be created for FAB feature.

## WHERE TO LOOK

| Task | File | Notes |
|------|------|-------|
| Add new data model | New file + `mod.rs` | Follow `asset_data.rs` as template |
| Change API client state | `mod.rs` | `Model` struct holds `RefCell<EpicGames>` |
| Add database table | `database.rs` + `migrations/` | Run `diesel migration generate` first |
| Modify favorite logic | `asset_data.rs` | Queries `favorite_asset` table |
| Change engine detection | `engine_data.rs` | Parses `Engine/Build/Build.version` JSON |
| Modify secret storage | `mod.rs` | `save_configuration()` — platform-specific |

## CONVENTIONS

- **GObject subclass pattern**: Every model has `mod imp` with `ObjectSubclass`, `ObjectImpl`, `ParamSpec` properties
- **RefCell interior mutability**: All fields wrapped in `RefCell<Option<T>>` or `RefCell<T>`
- **Signal emission**: Models emit `"refreshed"` or `"finished"` signals after async state updates
- **Async thumbnail loading**: `ProjectData` and `EngineData` spawn threads, send results via `async_channel`
- **Database access**: `crate::models::database::connection()` → r2d2 pool → `pool.get()` for each query
- **Property dispatch**: `set_property()`/`property()` match on `pspec.name()` string

## ANTI-PATTERNS

- `engine_data.rs:395` has **commented-out SSH key configuration** — incomplete feature, don't build on it
- **No validation** on GObject property setters — values accepted as-is
- Database schema is in `src/schema.rs` — **NEVER edit directly**, generated by Diesel

## DATABASE SCHEMA

Three tables (SQLite, bundled):
- `favorite_asset(asset TEXT UNIQUE)` — bookmarked asset IDs
- `user_data(name TEXT UNIQUE, value TEXT)` — key-value settings (display_name)
- `unreal_project_latest_engine(project TEXT UNIQUE, engine TEXT)` — project↔engine mapping
